# Contributing to Aethelgard

Welcome, Contributor! We appreciate your interest in enhancing the Aethelgard ecosystem. Whether you're fixing a logic bug, improving the UI, or adding new sanctuary features, your contributions are valuable.

## 🌿 Getting Started

1.  **Fork** the repository and **Clone** it to your local environment.
2.  Follow the **Installation** guide in the [README.md](README.md).
3.  Ensure your environment variables are configured correctly for local development.

## 🎋 Branching Strategy

To maintain a clean history, we use the following branch naming conventions:

- `feat/description-of-feature` — For new features.
- `fix/issue-description` — For bug fixes.
- `docs/what-was-updated` — For documentation changes.
- `refactor/what-was-improved` — For code improvements without functional changes.

## 🪵 Development Guidelines

### Code Style
- We use **Prettier** and **ESLint** for consistency.
- Keep components focused and reusable.
- Use **Tailwind CSS** for styling, adhering to the established color palette (Moss, Gold, Stone).

### Commit Messages
- Be descriptive but concise.
- Example: `feat: add unread badge to admin notification bell`

### Pull Request Process

1.  Create a branch from `main`.
2.  Implement your changes and test locally.
3.  Ensure no regressions occur in the booking or admin flows.
4.  Submit a Pull Request with a clear description of **what** changed and **why**.
5.  Link any related issues.

## 💬 Reporting Issues

Found a bug in the sanctuary?
Open an issue on GitHub with:
- A clear, descriptive title.
- Steps to reproduce.
- Expected vs. Actual behavior.
- Screenshots or console logs if applicable.

## ✅ PR Checklist

- [ ] Code is well-commented where necessary.
- [ ] No console logs or debug code left in production paths.
- [ ] Responsive design verified (Mobile/Tablet/Desktop).
- [ ] Accessibility (ARIs, alt text) considered.

Thank you for helping us build the future of luxury forest hospitality!
